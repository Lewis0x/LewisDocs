# 术语表

> 本表收录全系列高频出现的技术术语与缩写，按字母排序。每条 2-4 行定义 + 一条「深入」指向相关章节。
> 本页是**单一权威**——文中如术语第一次出现，链接 `[术语](/glossary#锚点)` 至此即可。

## A

### ACIS

Spatial Corp.（现为 Dassault Systèmes 子公司）出品的商业 B-Rep 几何内核。1989 年发布，是 SolidWorks（早期版本）、Bentley MicroStation、Autodesk AutoCAD 多个产品的内核选择。竞品：Parasolid、CGM、OCCT。
深入：[3.6 SolidWorks §一 历史演进](/platforms/solidworks#一、历史演进-30-年弧线)

### AcDb / AcGe / AcRx

Autodesk 自研类库前缀：**AcDb** = AutoCAD Database（数据库对象，如 `AcDbEntity`）；**AcGe** = AutoCAD Geometry（几何运算）；**AcRx** = AutoCAD Runtime（运行时）。三者构成 ObjectARX C++ SDK 的基础。
深入：[3.1 AutoCAD §三 对象模型](/platforms/autocad#三、对象模型-database-editor-document)

### APS（Autodesk Platform Services）

Autodesk 云端 API 集合，前身为 **Forge**（2015 年发布）。2022 年 12 月品牌重塑为 APS，覆盖 Design Automation / Model Derivative / Viewer / Data Management 等服务。
深入：[3.1 AutoCAD §七 Forge → APS](/platforms/autocad#七、forge-→-aps-云端生态)

### AutoLISP / Visual LISP

AutoCAD 嵌入式脚本语言，1986 年首发，至今仍可用。Visual LISP 是 1997 年发布的 IDE + 编译为 `.fas`/`.vlx` 的扩展。设计师友好层的早期典范。
深入：[3.1 AutoCAD §五 AutoLISP / Visual LISP](/platforms/autocad#五、autolisp-visual-lisp-30-年的设计师友好层)

## B

### B-Rep（Boundary Representation，边界表示）

3D 几何表示法：用顶点 / 边 / 面的拓扑连接 + 解析曲面（NURBS / 平面 / 圆柱面等）描述实体的边界。CAD 工业级建模的主流方案。
对比：Mesh-only（无拓扑）、Face-based（多边形 + 简单拓扑，如 SketchUp）。
深入：[文档 1 §2.1.1 几何表示的谱系](/theory#_2-1-1-几何表示的谱系)

### Briefcase

iTwin.js 中本地工作副本的称呼——一个 SQLite 文件，对应 iModelHub 上的某个 Changeset 时间点。客户端在 Briefcase 上编辑后，把 diff 推为新 Changeset。
深入：[3.5 MicroStation + iTwin §八 iTwin Platform](/platforms/microstation#八、itwin-platform-云原生重构)

---

## C

### CAA / RADE

**CAA**（Component Application Architecture）= CATIA 的 C++ 组件框架，包含 CATBaseUnknown 接口体系、TIE/BOA 扩展机制。**RADE**（Rapid Application Development Environment）= 配套的 Visual Studio 集成开发工具链 + mkmk 构建系统。
深入：[3.2 CATIA §二 CAA 体系架构](/platforms/catia#二、caa-体系架构概览)

### ChangeSet

iTwin.js 中"工作单元"的称呼——SQLite 行级 diff 的不可变快照。每个 ChangeSet 有 hash ID + 线性递增 index，构成 iModel 的版本时间线。Onshape 的对应概念是 microversion。
深入：[3.5 MicroStation + iTwin §八 iTwin Platform](/platforms/microstation#八、itwin-platform-云原生重构)

### CGM（CATIA Geometric Modeler）

Dassault Systèmes 自研的 B-Rep 几何内核，1998 年随 CATIA V5 上线。V5 / V6 / 3DEXPERIENCE 共享同一内核（A380 V4→V5 教训之后的回应）。竞品：Parasolid、ACIS。
深入：[3.2 CATIA §七 几何内核](/platforms/catia#七、几何内核-v5-v6-3dx-共享)

---

## D

### DAG（Directed Acyclic Graph，有向无环图）

参数化 CAD 中描述特征依赖关系的常见数据结构。每个特征是节点，"依赖于"是边。重算时按拓扑序执行。多数现代 CAD（SolidWorks、Inventor、NX 等）的特征关联性都基于 DAG。
深入：[文档 1 §2.2.3 特征关联性的传播机制](/theory#_2-2-3-特征关联性的传播机制)

### DGN

Bentley MicroStation 的原生文件格式。1979 年源自 Intergraph IGDS，2001 年 V8 版本切换到 OLE Compound File Binary 容器，是该格式最重要的现代化版本。
深入：[3.5 MicroStation + iTwin §一 历史演进](/platforms/microstation#一、历史演进-从-igds-到-itwin-的-40-年-api-时间线)

### DWG

AutoCAD 的原生文件格式。封闭格式，但 Open Design Alliance（ODA）通过逆向工程提供了开源读写库，被多数第三方 CAD 采用。AC1032 是 AutoCAD 2018-2026 的统一格式版本。
深入：[3.1 AutoCAD §一 历史演进](/platforms/autocad#一、历史演进-从-r12-到-2027-的-api-时间线)

---

## E

### ECObjects / ECSchema

Bentley 的工程元数据系统。**ECSchema** 用 XML 定义类、关系、属性、自定义特性；**ECObjects** 是运行时表达。是 iTwin 的 BIS 概念骨架，也是 MicroStation/iTwin 在工程数据集成场景的核心竞争力。
深入：[3.5 MicroStation + iTwin §六 ECObjects / ECSchema](/platforms/microstation#六、ecobjects-ecschema-bentley-独有的元元模型)

---

## F

### FeatureScript

Onshape 自研的几何编程 DSL（Domain-Specific Language），强类型、单位安全、运行在云端。Onshape 内置 feature（Extrude / Fillet 等）就是 FeatureScript 函数；标准库以开源 Onshape document 形式公开。
深入：[3.4 Onshape §五 FeatureScript](/platforms/onshape#五、featurescript-内嵌-dsl)

---

## G

### GRIP（Graphics Interactive Programming）

NX 的早期专有过程式编程语言，1980s Unigraphics 时代引入。语法接近 BASIC，被 NX Open（多语言现代 API）逐步替代，但仍作为遗产工作流保留。是"CAD 厂商各搞自家脚本语言"的典型遗产。
深入：[3.3 NX §一 历史演进](/platforms/nx#一、历史演进-从-uniapt-1972-到-nx-x-2024)

---

## I

### in-process

扩展模块加载到主程序进程内运行的执行模式（与 out-of-process 通过 IPC / RPC 通信相对）。性能高、可直接访问内部对象，但崩溃会拖垮主程序。AutoCAD ObjectARX、CATIA CAA、SolidWorks SDK、MicroStation MDL/COM 都选择此模式；Onshape 等纯 REST 平台则属于 out-of-process。
深入：[文档 2 §2.2 API 入口对象对比](/comparison#_2-2-api-入口对象对比)

### IFC（Industry Foundation Classes）

buildingSMART International 制定的 BIM 数据交换开放标准（ISO 16739），用 EXPRESS schema 描述建筑、构件、空间、关系。AEC 行业事实标准。
深入：[3.5 MicroStation + iTwin](/platforms/microstation)

### iModel / iModelHub

Bentley 数字孪生数据载体（**iModel**）+ 云端版本管理服务（**iModelHub**）。iModel 物理上是 SQLite + ECSchema；iModelHub 维护 ChangeSet 时间线 + Briefcase 分发。
深入：[3.5 MicroStation + iTwin §八 iTwin Platform](/platforms/microstation#八、itwin-platform-云原生重构)

---

## K

### KBE（Knowledge Based Engineering，基于知识的工程）

把工程规则、设计意图、参数关系编码为可执行知识库的设计方法。CATIA 的 Knowledge Advisor、NX 的 Knowledge Fusion 都属于 KBE 工具。
深入：[3.3 NX §五 Knowledge Fusion](/platforms/nx#五、knowledge-fusion-参数化规则建模)

### Knowledge Fusion（KF）

Siemens NX 的规则建模 / KBE 引擎，类 Lisp 语法，支持"自适应持久对象"（Adaptive Persistent Object）。用于工程规则、参数化、自动化建模。
深入：[3.3 NX §五 Knowledge Fusion](/platforms/nx#五、knowledge-fusion-参数化规则建模)

## M

### MDL（MicroStation Development Language）

Bentley MicroStation 的扩展开发框架。1990s 初的 V4 版本以 C 语法子集 + p-code 字节码诞生（一次编译多平台运行）；从 V8 起转变为 C 库（可编译为 Windows DLL）。2024 起 Python API 出现。
深入：[3.5 MicroStation §一 历史演进](/platforms/microstation#一、历史演进-从-igds-到-itwin-的-40-年-api-时间线)

### microversion

Onshape 数据模型的最小变更单元——每次编辑产生一个不可变 microversion，存储父引用 + definition change。等价于 git commit。Onshape 的 git 式四层数据模型的基础。
深入：[3.4 Onshape §三 核心数据模型](/platforms/onshape#三、核心数据模型-git-式四层)

---

## N

### NURBS（Non-Uniform Rational B-Splines）

非均匀有理 B 样条曲线 / 曲面。CAD 中描述自由曲面的标准数学表示，能精确表达圆、椭圆、抛物面等所有圆锥曲线。B-Rep 内核的曲面建模基础。
深入：[文档 1 §2.1.1 几何表示的谱系](/theory#_2-1-1-几何表示的谱系)

---

## O

### ObjectARX

AutoCAD Runtime eXtension，1995 年随 R13 发布的 C++ 原生扩展架构。`.arx` 模块本质是 Windows DLL，进程内合作 / 直接访问内部类。是 AutoCAD 30+ 年 API 哲学的基石。
深入：[3.1 AutoCAD §二 API 整体架构](/platforms/autocad#二、api-整体架构-六层金字塔)

### ODA（Open Design Alliance）

非营利的 CAD 文件格式逆向工程联盟，1998 年成立（前身 OpenDWG Alliance）。最重要的产出是 **Teigha（现 ODA SDK）**——开源 / 商业双轨的 DWG / DGN 读写库，让第三方 CAD（FreeCAD、BricsCAD、ZWCAD 等）能读写 Autodesk 与 Bentley 的封闭格式。
深入：[3.1 AutoCAD §9.4 "DWG 是堡垒"](/platforms/autocad#_9-4-dwg-是堡垒)

### Observer（观察者模式）

经典 GoF 设计模式：被观察对象在状态变化时主动通知所有注册的观察者，解耦事件源与响应方。CAD 平台普遍使用：AutoCAD 的 Reactor、CATIA 的 CallBack、NX 的 Subscriber、SolidWorks 的 EventHandler、SketchUp 的 17+ 类 Observer、FreeCAD 的 boost::signals2。是扩展无侵入地监听 CAD 内部状态的统一抽象。
深入：[文档 2 §3.4 模式 D — 事件 / 观察者机制](/comparison#_3-4-模式-d-事件-观察者机制)

### OCCT（Open CASCADE Technology）

开源 B-Rep 几何内核，2000 年 Open CASCADE SAS 开源（最初 GPL，2014 年起 LGPL）。FreeCAD、KiCad 3D viewer、Salome 等开源 CAD 的几何引擎；商业 CAD 如部分国产 CAD 也采用。
深入：[3.8 FreeCAD §八 与 OCCT 的关系](/platforms/freecad#八、与-occt-的关系-开源-cad-的几何基石)

## P

### Parasolid

Siemens（前身 Unigraphics）出品的商业 B-Rep 几何内核，1988 年发布。**SolidWorks / Solid Edge / NX / Onshape 都以它为内核**——是 CAD 业界使用最广的"内核中立"商业产品。
深入：[3.6 SolidWorks §一 历史演进](/platforms/solidworks#一、历史演进-30-年弧线)

### PDM（Product Data Management）/ PLM（Product Lifecycle Management）

**PDM** = 产品数据管理，文件级 check-in/check-out + 元数据 + 权限。**PLM** = 在 PDM 之上加产品生命周期（从概念到退役）的流程管理。代表：Teamcenter（Siemens）、ENOVIA（DS）、Windchill（PTC）。
深入：[文档 1 §2.2.2 文档的两种存储范式](/theory#_2-2-2-文档的两种存储范式)

---

## S

### Synchronous Technology

Siemens NX 5（2007）首次引入的混合建模技术：直接编辑几何（移动 / 旋转 / push-pull）的同时**自动识别并维护几何关系**（同心 / 共面 / 对称 / 相切等）。让 history-based 与 history-free 路线在同一平台共存。
深入：[3.3 NX §四 Synchronous Technology](/platforms/nx#四、synchronous-technology-直接编辑-关联性)

---

## T

### TNP（Topological Naming Problem，拓扑命名问题）

参数化 CAD 中的根本性挑战：当上游特征参数变化时，下游引用的 face / edge / vertex ID 可能变化或消失，导致模型断裂。商业 CAD 通过启发式 + 用户重选缓解；FreeCAD 1.0 用 ElementMap 拓扑历史哈希；Onshape 用 Query 语言路线规避。
深入：[文档 1 §2.1.3 拓扑命名问题](/theory#_2-1-3-拓扑命名问题-topological-naming-problem-tnp)
