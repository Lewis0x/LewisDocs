# 使用文档

> 本文档面向**站点的阅读者**——团队成员、ISV、外部评审人。
> 如果你是开发 / 维护这个站点的人，请看 [03-开发文档](./03-development.md)。

## 1. 如何访问

部署后可通过以下入口访问（具体 URL 询问站点维护者）：

- GitHub Pages（默认）：`https://<org>.github.io/<repo>/`
- GitLab Pages（内网部署）：`https://<group>.gitlab.io/<project>/`
- 内网 Nginx：`https://docs.your-company.cn/cad-api/`
- 公司 OSS / COS 静态托管的内网域名

> 站点是**纯静态**的——浏览器加载完后所有交互都在本地，关闭服务器也能用浏览器缓存继续看。

## 3. 阅读路径建议

首页（导读）有四条**预设阅读路径**，选适合自己角色的入口：

| 路径 | 适合人群 | 用时 | 入口 |

| A | 决策者 / CTO | 30 min | 文档 1 决策框架 + 文档 2 选型建议 |

## 4. 导航

### 4.1 顶部导航

页面顶部的菜单条提供主入口：

- **首页** —— 系列导读
- **理论框架** —— 文档 1
- **横向对比** —— 文档 2
- **厂商深度** —— 下拉到具体厂商

### 4.2 左侧侧边栏

每页左侧的目录树覆盖**整个系列**，分两组：

- **系列总览**：导读、理论、对比
- **厂商深度剖析**：3.1 - 3.8

侧边栏有「折叠 / 展开」按钮，移动端会自动收起。

### 4.3 右侧大纲

每页右侧自动列出**当前文档的 H2 / H3 标题**——相当于本页的"目录"。点击直接跳到对应章节。

### 4.4 文档底部翻页

页面最下方有 **上一篇 / 下一篇** 导航，按侧边栏顺序遍历整个系列。

## 5. 搜索

### 5.1 调出搜索框

- 点击页面右上角搜索图标
- 或按 `Ctrl + K`（Windows / Linux） / `⌘ + K`（macOS）

### 5.2 搜索能力

- **全文中文搜索**：基于本地 MiniSearch 索引 + 浏览器原生 `Intl.Segmenter` 中文分词
- **跨文档**：一次搜索覆盖全部 11 篇文档
- **模糊匹配**：`fuzzy: 0.2`——少量错字也能命中
- **前缀匹配**：输入 "几何" 能命中 "几何内核"、"几何表示" 等

### 5.3 搜索技巧

| 想找 | 输入 |
|---|---|
| 几何相关讨论 | `几何内核` / `B-Rep` / `OCCT` |
| 拓扑命名问题 | `TNP` / `拓扑命名` / `ElementMap` |
| 参数化 vs 直接编辑 | `synchronous` / `直接编辑` |
| 云协作 | `Onshape` / `git 式` / `ChangeSet` |
| AI 整合 | `AI` / `Aura` / `2026` |
| 决策框架 | `12 维` / `难撤回决策` |

> 搜索结果按相关度排序：标题命中权重最高，正文次之。

## 6. 跨文档跳转（回链）

研究文档大量使用 **`[回链：3.x §一 …]`** 形式的交叉引用，本站把 40 处回链全部改写为可点击链接：

- **单目标**：`[回链：3.4 §三 核心数据模型]` → 一个蓝色链接，点击跳到 3.4 文档对应章节
- **多目标**：`[回链：3.2 §六 X；3.3 §三 Y]` → 多个相邻链接，分别可点

> **悬停可预览**：在链接上悬停几秒，VitePress 会预加载目标页面。

## 7. URL 与书签

每个章节都有**独立、稳定的 URL**：

```
首页              https://<host>/
理论框架          https://<host>/theory
横向对比          https://<host>/comparison
Onshape 厂商页    https://<host>/platforms/onshape
Onshape 第三章    https://<host>/platforms/onshape#三、核心数据模型-git-式四层
```

这些 URL **可以放心保存到书签或贴到 IM 分享**。维护团队承诺在 V1.x 内不会改动这些 URL（详见 [02-方案文档 §9](./02-design.md)）。

> **章节 URL 规则**：
> - 厂商 3.x 的二级标题用 `#中文数字、章节名` 形式（如 `#三、核心数据模型-git-式四层`）
> - 理论文档的小节用 `#_章节号-章节名`（首位数字会加 `_` 前缀）

## 8. 深色模式

页面右上角有 **🌞 / 🌙** 切换按钮：

- 浅色：白底黑字，长时间阅读 / 截图友好
- 深色：暗底亮字，夜间 / OLED 屏护眼

切换会保存到浏览器 LocalStorage，下次访问保持。

> 默认跟随系统：如果你的操作系统设了深色模式，站点初次加载就是深色。

## 9. 移动端阅读

- **响应式布局**：手机宽度下侧边栏自动收起，左上角点击展开
- **可读字号**：默认行高 1.8，中文段落舒服
- **触摸滑动**：表格在窄屏下可横向滚动

> 横屏阅读体验更好，特别是看横向对比表格的时候。

## 9bis. 架构图与时间线

每篇厂商深度剖析（3.1 - 3.8）开头都有一张 **timeline** 图，展示该平台 30-50 年的关键演进；接着的「整体架构」章节有一张 **架构层级图** 展示 API 各层关系。

理论框架（文档 1）有 4 张核心图：

- 四大支柱图（§1.2）
- 几何精度谱系（§2.1.1）
- 扩展数据 Level 0-5 光谱（§2.3.1）
- 协作模式 1-6 演进（§2.4.2）

**这些图都是 Mermaid 渲染的真实 SVG**——可以：

- **缩放**：浏览器 `Ctrl + 滚轮` 不会糊，文字始终清晰
- **跟随主题**：浅色 / 深色模式自动切换配色
- **复制图源**：右键查看页面源代码，找到对应 ` ```mermaid ` 块即可复制图代码到 [mermaid.live](https://mermaid.live/) 二次编辑
- **打印 / 截图**：因为是 SVG，截图后放进 PPT / Word 不会失真

> 如果你看到图位置只显示一段文本而非图形，说明浏览器 JS 被禁或网络拦截了 Mermaid CDN——刷新或换一个网络环境试试。

## 10. 标记符号速查

研究文档里反复出现以下标记，理解它们能更快读懂：

| 符号 | 含义 |
|---|---|
| ⚠️ | 警告 / 重要前置说明 |
| ⭐ | 关键观察 / 推论的"高显著度"标注 |
| `> [判断]` | 作者的主观判断（已附证据等级 A/B/C） |
| `> [评论]` | 作者的旁注 / 思辨 |
| `> [推论]` | 从证据出发的逻辑推断 |
| `> [陷阱]` | 实施时容易踩的坑 |
| `[证据等级 A]` | 多源验证 / 高确信 |
| `[证据等级 B]` | 单源 / 中等确信 |
| `[证据等级 C]` | 推断 / 低确信 |

详见首页**导读**与文档 1 的「阅读约定」章节。

## 12. 常见疑问

**Q1：搜「面」搜不到东西？**
单字搜索因为分词器倾向"词"，建议输入完整术语。试 `B-Rep 面`、`选面` 等。

**Q2：回链点击没反应？**
检查浏览器是否禁了 JS。VitePress 单页应用需要 JS 支持页面切换。

**Q3：能离线看吗？**
站点纯静态，理论上保存为 HTML 就能本地打开。但搜索 / 路由依赖 JS，不联网时主体内容仍可读。

**Q4：能导出 PDF 吗？**
浏览器自带打印 → 另存为 PDF 即可。一篇文档约 30-60 页 A4。

**Q5：能在企业微信 / 钉钉 / 飞书里直接预览吗？**
看具体 IM 是否支持自定义域名预览。多数情况只能贴链接，对方点开浏览器查看。

**Q6：页面看起来很简朴，能不能更花哨？**
不能。请见 [01-需求文档 §5](./01-requirements.md)：「克制研究稿气质」是显式需求。

**Q7：发现一处文档错误，如何修？**
不要直接改站点上看到的内容。源是 V4 上游 Markdown，要去那里改，再走团队的发布流程。
